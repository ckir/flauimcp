# docs-spec — flaui-mcp documentation convention

The contract the `docs-rationalize` skill binds to. Human-facing docs are role-scoped and written in a
terse, human, scannable voice. (Fuller rationale: `docs/superpowers/specs/2026-07-18-docs-rationalization-design.md`.)

## Docs (audience → voice)

| Doc | Audience | Voice |
|---|---|---|
| `README.md` | Orient + get running fast, then route out (≤ ~90 lines) | terse-technical |
| `docs/operator-manual.md` | The human who installs, configures, runs, and audits the daemon | terse-technical |
| `docs/agent-contract.md` | The RPC surface an agent (or a human debugging one) consumes | terse, schema-dense (may be long) |
| `docs/architecture-and-safety.md` | How it works and why it's safe — guarantees + rationale | terse-technical |
| `docs/building.md` | A contributor building the server or cutting a release | terse-technical |
| `CONTRIBUTING.md` | A new contributor | terse-technical |
| `.github/pull_request_template.md` | A PR author's checklist | terse |

## Voice / scannable rules

- Terse-technical and human — short lines, no run-on sentences, no AI-tells.
- Scannable: headings, tight fenced code blocks, small tables, bullet lists ≤ 5 items; prose reserved for reasoning.
- Prefer a code block / command over prose where it carries the meaning.
- `agent-contract.md` is exempt from brevity — it is a schema-dense lookup reference.

## Do-not-touch (out of scope for a docs pass)

- `LICENSE`, `CLA.md` — legal text.
- `CHANGELOG.md` — Keep-a-Changelog format, managed by `scripts/release.ps1`.
- `docs/superpowers/**` — specs and plans (working artifacts, not product docs).
- Generated: `dist/**`, `plugins/flaui-mcp/skills/**` (regenerated by `scripts/build-plugin.ps1`).
- Any code or workflow file (`*.cs`, `*.ps1`, `*.yml`) — a docs pass never edits the system it describes.
