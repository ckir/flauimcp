# v0.11.2 — Session Hygiene (minimize orphan fix) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop `desktop_window_transform minimize` from orphaning the human's keyboard when it minimizes the foreground window, generalize the v0.11.1 close healer so close + minimize share it, and add a `foregroundGained` signal to `desktop_focus_window`.

**Architecture:** The v0.11.1 healer (`WindowManager.RestoreForegroundAfterClose`) is generalized to `RestoreForegroundAfterCollapse(hwnd, hasCollapsed)` — the only difference between close and minimize is the "collapsed" tell (close = window destroyed `!IsWindow`; minimize = window yielded foreground `GetForegroundWindow()!=hwnd`). A new `WindowManager.WindowTransformAsync` wraps `Interactor.WindowTransform` with foreground capture-before / restore-after on the minimize case, keeping all Win32 in `WindowManager` where it already lives (no new class — the "shared guard" is the shared private method). Scope A (minimal) from the SP1 spec; SP3 harness + SP4 sentinel deferred.

**Tech Stack:** C# / .NET 10, FlaUI/UIA3, xUnit. Build: `dotnet build`; headless tests: `dotnet test --filter "Category!=Desktop"`; Desktop tests run at a real console only (headless/RDP box can't drive them).

**Spec:** `docs/superpowers/specs/2026-07-05-flaui-mcp-session-hygiene-sp2-harden-design.md` (+ SP1 audit sibling). **Oracle for correctness:** SP2 §2.1–§2.4 + the v0.11.1 close behavior (the minimize fix must mirror it).

```csharp
/// <summary>Best-effort keyboard-focus restoration after the OS-foreground window COLLAPSES (is closed
/// or minimized). The Win32 foreground-lock only relaxes once the window stops being a valid foreground,
/// so spin-wait (bounded) for <paramref name="hasCollapsed"/> before claiming foreground for the next
/// Z-ordered top-level. Runs on the caller's STA; pure Win32 (no UIA). NEVER throws — the transform has
/// already succeeded, so a failed refocus must not surface as an error. No AttachThreadInput
/// (deadlock-prone against a thread hanging during shutdown).</summary>
private void RestoreForegroundAfterCollapse(IntPtr collapsedHwnd, Func<bool> hasCollapsed)
{
    try
    {
        var deadline = DateTime.UtcNow.AddMilliseconds(500);
        while (!hasCollapsed() && DateTime.UtcNow < deadline) Thread.Sleep(25);
        // No-steal guard (AGY-AFTER plan-review): only restore if we are ACTUALLY orphaned — foreground
        // is null, or still the collapsing window. If some OTHER window already holds it (human clicked
        // away mid-wait, or the OS reactivated a good window), leave it — never yank focus off the
        // human's deliberate choice.
        var current = GetForegroundWindow();
        if (current != IntPtr.Zero && current != collapsedHwnd) return;
        var next = PickForegroundFallback(EnumTopLevel(), collapsedHwnd);
        if (next != IntPtr.Zero) SetForegroundWindow(next);
    }
    catch { /* focus restoration is best-effort; the window is already collapsed */ }
}
```

**Note (AGY-AFTER plan-review, finding 3):** the no-steal guard is a deliberate enhancement over the
v0.11.1 healer — close *and* minimize now heal only a true orphan and never steal focus the human just
chose. It still passes the v0.11.1 close smoke (orphan → `current` is null/dead → guard passes →
fallback set; non-orphan → guard returns, correctly, because there was nothing to fix). Because this
slightly changes the shipped close path, re-run the v0.11.1 close smoke as part of the v0.11.2 console
smoke.

- [ ] **Step 1: Add the method.** It captures foreground state BEFORE the transform and restores AFTER, but ONLY for `minimize` (maximize/restore keep the window foreground → no orphan):

```csharp
/// <summary>Run a window transform (maximize|minimize|restore) with session hygiene: minimizing the
/// OS-foreground window orphans keyboard focus the same way closing it does (background process +
/// foreground-lock), so capture foreground ownership BEFORE and restore it AFTER via the shared healer.
/// maximize/restore keep the window foreground, so they are pass-through.</summary>
public Task<bool> WindowTransformAsync(WindowHandle handle, string action, int timeoutMs) =>
    RunOnWindowActionAsync(handle, (winEl, _) =>
    {
        var w = winEl.AsWindow();
        bool minimizing = string.Equals(action?.Trim(), "minimize", StringComparison.OrdinalIgnoreCase);
        IntPtr hwnd = IntPtr.Zero;
        bool wasForeground = false;
        if (minimizing)
        {
            try { hwnd = w.Properties.NativeWindowHandle.ValueOrDefault; } catch { }
            wasForeground = hwnd != IntPtr.Zero && GetForegroundWindow() == hwnd;
        }
        Interactor.WindowTransform(w, action);
        // A minimized window still exists (IsWindow stays true), so the "collapsed" tell is that it has
        // yielded the foreground — NOT !IsWindow.
        if (minimizing && wasForeground)
            RestoreForegroundAfterCollapse(hwnd, () => GetForegroundWindow() != hwnd);
        return true;
    }, timeoutMs);
```

**Note (AGY-AFTER plan-review, finding 4):** unlike close (which runs the healer on the *query* STA),
this runs on the *action* STA, so the ≤500ms bounded wait briefly serializes behind/ahead of other
*mutations* rather than reads. Benign (a terminal transform op), and arguably better — it does not
starve independent read-queries. Documented so it is not a silent behavioral shift.
