## Part 1: Open Item Answers

1.  **Antigravity (agy) Configuration:**
    * **File:** Antigravity typically uses `~/.gemini/settings.json`. While CLI-specific overrides exist, mutating the base file is the most reliable path for broad toolchain compatibility.
    * **Permissions:** Appending `"mcp(flaui-mcp/*)"` to `permissions.allow` is both required and sufficient.
    * **Restart:** Mandatory. The agent must be restarted to re-initialize the tool registry.
    * **Confidence:** High.

2.  **Claude Code Path Resolution:**
    * **Interpolation:** Claude Code **does not** support `${CLAUDE_PLUGIN_ROOT}` expansion.
    * **Implementation:** Use relative paths. The client resolves paths relative to the directory containing the `.mcp.json` file. The `command` should invoke the shell explicitly: `"command": "powershell", "args": ["-ExecutionPolicy", "Bypass", "-File", "./bin/flaui-mcp-launch.ps1"]`.
    * **Confidence:** High.

## Part 2: Bootstrapper Stress Test (Failure Modes)



The bootstrapper pattern introduces critical risks to the MCP runtime:

* **Stdio Corruption (Severity: CRITICAL):** PowerShell forces line-ending translations (`\n` to `\r\n`) and adds Byte Order Marks (BOM), which will cause JSON-RPC framing errors in the agent. Ambient PowerShell output (module warnings) will corrupt the stream. **Recommendation:** Use a native shim or a simple `.cmd` batch file to `call` the binary directly.
* **Startup Timeout (Severity: HIGH):** Fetching an 80MB binary on first-run will likely exceed MCP `initialize` request timeouts (typically 10-30s). **Recommendation:** Move download logic outside the MCP lifecycle or provide explicit user-facing log output.
* **Execution Policy (Severity: HIGH):** Default Windows settings block `.ps1` execution. Corporate AppLocker often prohibits scripts in `%LOCALAPPDATA%` entirely.
* **Concurrency/Races (Severity: MEDIUM):** Simultaneous launches will cause file-lock collisions. **Recommendation:** Download to a `.tmp` file and perform an atomic `Move-Item -Force`.

## Part 3: Uninstall & Update Gaps

* **Config Backup/Restore Trap (Severity: HIGH):** Restoring a backup on uninstall will wipe any user configuration changes made between installation and uninstallation. **Recommendation:** Perform targeted JSON removal instead of restoring old files.
* **File Locking (Severity: HIGH):** Updates will fail if the server is running because the binary is locked by the OS. **Recommendation:** The installer must attempt to terminate the process before updating.
* **Orphaned Binaries (Severity: MEDIUM):** Removing the Claude plugin does not trigger the `flaui-mcp` uninstaller, leaving the 80MB binary and stale configuration permanently on the system. **Recommendation:** Clear user documentation requiring manual uninstallation via CLI before plugin removal.
